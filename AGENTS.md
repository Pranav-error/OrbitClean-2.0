# OrbitClean 2.0 — Agent Rules

This file governs how AI coding agents (Claude Code, Codex, Cursor, etc.) should work in this repository. Read it fully before making any changes.

---

## Understand the Codebase First

- Read `CLAUDE.md` for the full project overview, tech stack, and architecture before writing any code.
- Read the relevant source file before editing it. Never make changes based solely on file names.
- The frontend seed data lives in `frontend/src/lib/data.ts`. The real data comes from the FastAPI backend at `http://localhost:8000`. Do not confuse them.

---

## Language & Framework Conventions

### Python (ml/, backend/)

- Python 3.10+. Use type hints on function signatures.
- No f-strings for SQL. No shell injection via `subprocess` with user-controlled strings.
- All ML scripts must support a `--demo` flag that produces valid output without real satellite data or external services.
- Fallbacks: if an optional dependency (XGBoost, Prophet, torch) is missing, log a warning and use the scikit-learn fallback. Never raise an unhandled ImportError in production paths.
- Return consistent JSON schema from every API endpoint, even in error or fallback states.
- Do not hardcode file paths. Use `pathlib.Path(__file__).parent` relative paths.

### TypeScript / Next.js (frontend/)

- Next.js 14 App Router. Use Server Components by default; add `"use client"` only when interactivity or browser APIs are required.
- All API calls go to `http://localhost:8000` (backend). Use the `NEXT_PUBLIC_API_URL` env var if present, otherwise default to that.
- TypeScript strict mode is on. Fix type errors — do not use `any` unless there is no alternative.
- Tailwind CSS for styling. Do not add additional CSS frameworks.
- Leaflet maps must be dynamically imported (`next/dynamic` with `ssr: false`) to avoid SSR window errors.
- Components that fetch data should handle loading and error states.

---

## Data File Rules

- **Never delete** `data/detected_dumps.geojson` or `data/risk_grid_predicted.geojson` — they are pre-computed ML outputs used by the backend at startup.
- **Never commit** `.SAFE` Sentinel-2 archives, raw `.tif` raster files, or files >50 MB.
- GeoJSON coordinate order is `[longitude, latitude]` (GeoJSON spec). Do not swap.
- All GeoJSON uses WGS84 (EPSG:4326). Do not reproject unless specifically asked.
- `data/cleanup_missions.json` and `data/anomaly_alerts.json` are runtime-mutated — do not treat them as static.

---

## API & Backend Rules

- All endpoints return `{"error": "...", "detail": "..."}` on failure — not raw exceptions.
- CORS is configured in `backend/app.py`. Do not add new origins without checking existing config.
- The `/api/routes/static` endpoint is the demo-safe fallback — do not modify its schema.
- The `/api/query` endpoint calls Claude API when `ANTHROPIC_API_KEY` is set; falls back to mock responses otherwise. Both paths must return the same JSON schema.
- Do not add new Python dependencies to the backend without updating `requirements.txt`.

---

## ML Model Rules

- Each ML script in `ml/` is standalone and importable by `backend/app.py`. Keep them decoupled.
- The 3-layer fallback in `ml/route_optimizer.py` (`full → estimation → static`) must never be broken. All three layers must return identical JSON schema.
- `ml/community_validator.py` defines the 3-report / 200m verification threshold. Do not change these constants without updating the README and `backend/app.py` accordingly.
- `ml/auto_retrain.py` is triggered at 50 community photos. This threshold is defined in `backend/app.py` — keep them in sync.
- Carbon credit formula constants (IPCC Tier 1) in `ml/carbon_estimator.py` are fixed scientific values. Do not adjust them.

---

## What NOT to Do

- Do not add `console.log` / `print` debug statements to committed code.
- Do not introduce new state management libraries (Redux, Zustand, etc.) — the app uses React state and prop drilling intentionally.
- Do not change the WAScore formula in `backend/ward_scorer.py` without updating the formula in `CLAUDE.md` and `README.md`.
- Do not rename GeoJSON property keys in data files (e.g., `risk_score`, `dump_id`, `area_sqm`) — the frontend and backend both depend on these exact names.
- Do not add authentication or session management — this is a public dashboard.
- Do not modify `data/route_solution.json` manually — it is generated by `ml/route_optimizer.py`.

## Testing Approach

- There is no automated test suite. Validate changes by running:
  ```bash
  # Backend — check all endpoints respond
  uvicorn backend.app:app --port 8000 --reload
  curl http://localhost:8000/api/summary
  curl http://localhost:8000/api/dumps/active
  curl http://localhost:8000/api/routes/static

  # Frontend — check dashboard loads and map renders
  cd frontend && npm run dev
  # Open http://localhost:3000 and verify each tab renders without console errors
  ```
- For ML scripts, run with `--demo` flag and check that output GeoJSON is valid.
- TypeScript errors: run `cd frontend && npx tsc --noEmit` before reporting a frontend change as complete.

## Commit Hygiene

- Commit messages should be imperative, lowercase, under 72 chars.
- Do not commit secrets (`ANTHROPIC_API_KEY`, Twilio credentials, etc.).
- Do not commit large binary files (`.tif`, `.SAFE`, `.pptx`). These are already gitignored.

## SWM Rules 2026 Compliance Context

Any feature touching waste classification must respect the mandatory 4-stream model:

| Stream | Bin colour | Contents |
|--------|-----------|----------|
| Wet | Green | Kitchen waste, food scraps |
| Dry | Blue | Paper, plastic, glass, metal |
| Sanitary | Red | Diapers, sanitary pads, tissues |
| Hazardous | Black | Paint, batteries, CFL bulbs, medicines |

The classifier output from `ml/classifier_api.py` maps to these exact stream names. Do not add or rename streams.
